# Data Verification Policy

## Verification Statuses

| Status | Meaning |
|--------|---------|
| verified | Confirmed by editor with source and date |
| community | User-submitted, pending or partial review |
| estimated | Derived from nearby data or general knowledge |
| demo | Demonstration data — NOT for production |
| unknown | Insufficient data |

## Rules

1. Demo seed data must never be presented as legally verified
2. AI must not claim verification without database support
3. Regulation records require source, effective date, and last checked date
4. Stale data (>12 months) shows warning badge in UI
5. Community reports require moderator approval before affecting public records

## Display Requirements

Every spot detail page shows:
- Verification badge
- Last verified date (when available)
- Source list with reliability level
